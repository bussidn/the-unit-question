# Part 3 : Mocks Clean

## Contexte

Vous héritez d'une codebase avec un `OrderCancellationService` et ses tests.

Les tests ont été écrits par un développeur qui "sait ce qu'il fait" :
- Mocks propres avec MockK
- `verifySequence` pour s'assurer de l'ordre des opérations
- Bonne couverture

**Tous les tests passent.** ✅

---

## Votre Mission

1. **Lisez le code** — `OrderCancellationService.kt`
2. **Lisez les tests** — `OrderCancellationServiceTest.kt`
3. **Lancez les tests** — `./gradlew test`
4. **Répondez :** Ces tests sont-ils suffisants ? Le code est-il correct ?

---

## Temps

~10 minutes

---

## Discussion

Après votre analyse, discutez avec le groupe :

- Quels scénarios sont testés ?
- Quels scénarios NE SONT PAS testés ?
- Feriez-vous confiance à ce code pour aller en production ?
