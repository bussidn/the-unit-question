# The Unit Question

> Qu'est-ce qu'un "unit" dans unit test ?

## 🎯 À propos

Ce repository contient le matériel pour un atelier (ou une présentation) sur les heuristiques de test.

**Le sujet** : La notion de "unit" dans les tests unitaires est galvaudée. Chacun y met sa propre définition. Ce workshop vous fait vivre différentes approches de test pour découvrir par vous-même laquelle vous convient.

## 🧭 Déroulé de l'atelier

L'atelier s'appuie sur un même domaine métier : **une petite boutique en ligne**. On y retrouve un code de production centré sur la prise de commande, le paiement, le stock, l'expédition et les notifications.

L'objectif n'est pas de découvrir un nouveau domaine à chaque exercice, mais au contraire de **rester dans un univers fonctionnel stable** pour mieux ressentir l'impact des choix de test.

Dans chaque langage, l'atelier se déroule en **4 parties qui s'enchaînent dans l'ordre**. Chaque partie vous demande d'implémenter **la même feature** — mais avec une boîte à outils de test différente. Vous pouvez ainsi comparer directement comment chaque approche influence l'écriture, la lecture et la maintenance des tests.

> 💡 **Astuce** : Le code de production est le même d'une partie à l'autre. Une fois la feature implémentée dans une partie, vous pouvez copier votre code de production dans la suivante et vous concentrer uniquement sur l'approche de test.

Les consignes détaillées, les scénarios métier et la feature à implémenter se trouvent dans le `README` de chaque partie.
